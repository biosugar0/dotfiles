# Orchestrator

複雑タスクを順次ステップに分解し、各ステップ内で並列サブタスクを実行する。

Execute `/orc` slash command for detailed workflow.

## 発動条件

- 3つ以上の独立したサブタスクが識別できる
- 依存関係のある複数ステップが必要
- 並列実行で効率化できる

## 基本フロー

```
[分析] → [ステップ計画] → [並列実行] → [レビュー・適応] → [次ステップ]
```

## コンテキスト効率化

- 各サブタスクに最小限のコンテキストを渡す
- 結果は100-200語の要約で受け取る
- 親コンテキストを圧迫しない
