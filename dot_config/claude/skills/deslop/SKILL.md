# AI Code Slop 削除

コード変更完了後、mainブランチとのdiffをチェックし、このブランチで導入されたAI生成のslopを削除する。

## 対象となるslop

- **余計なコメント**: 人間が書かないようなコメント、ファイル内の他のコメントと一貫性がないもの
- **過剰な防御的チェック**: そのコード領域では異常なtry/catch、特に信頼済み/バリデーション済みのコードパスから呼ばれる場合
- **一貫性のないスタイル**: ファイル全体のスタイルと合わないコード

## 実行手順

1. mainブランチとのdiffを取得
2. 各変更ファイルを分析
3. slopを特定して削除
4. 変更内容を1-3文で要約報告

## 注意事項

- 機能的な変更は行わない（slopの削除のみ）
- 既存のコードスタイルに合わせる
- 不明な場合は削除しない（保守的に判断）
