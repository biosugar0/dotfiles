- Opus 4.8 には tool-call タグ破損("court"化・text 漏洩)の既知バグがある。実測で全ツール(Bash/Edit/Read/Write/Agent)・ペイロード長非依存に起きるため、コマンド単位の委譲では防げない(1 Bash→1 Agent の置換は tool-call 回数を減らさない)。Bash の sonnet-bash-runner 委譲は「複数コマンドをまとめて任せる」粒度か、破損発生後のリトライ先としてのみ使う
- 破損を検知/指摘されたら: 次の応答は前置きテキストを一切書かず先頭トークンから tool call を出し直す。長大な verbatim 再送・インライン heredoc は再破損しやすい(スクリプトファイルか bin/ ヘルパー経由に)
- 破損は連鎖する: 壊れた出力が履歴に残るとモデルがそれを手本に模倣して再発率が上がる(self-poisoning)。同一セッション累計3回目以降は粘らず、ユーザーに「exit して claude --resume で再開」を提案せよ(SessionEnd hook が破損履歴を自動無害化するため、文脈を保ったまま毒を抜ける。代替: /model sonnet、全損してよければ /clear)
- 漏洩 XML の生タグを text で引用・例示しない(履歴に手本を植えると自己汚染で悪化する)。言及は「court化」「タグ破損」等の言い換えで行う
