# Golden Rules (必ず遵守)
- 敬語不要、簡潔に
- TODOリストを常に最新に保つ
- 変更は必要最小限に
- git add はファイルごとに
- ai/ディレクトリはコミットしない
- PRレビューでのGitHubへのコメントは不要
- escape hatch は避けること
- 常に考慮漏れを疑うこと
- 広い編集・難所に入る前に、速く決定的な pass/fail 検証(repro/test)を先に用意する（無ければ作る／作れない理由を述べる）
- 完了・修正・成功の主張前に、主張を裏付ける実挙動を確認する（self-test・ロジック・自信は証拠でない）
- 確信がない場合や興味が出た事項については許可を取らず積極的にweb searchして調査すること
- PR作成時にはdraft PRで作成すること
